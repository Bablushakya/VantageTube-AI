WHAT WAS FIXED
================================================================================

The YouTube OAuth connection was failing with a CORS error. The issue was that
the frontend was using fetch() to call the OAuth endpoint, which caused the
browser to block the cross-origin redirect to Google.

SOLUTION: Changed the backend OAuth endpoint to accept user_id as a query
parameter instead of requiring an authentication header. This allows the
frontend to use direct navigation (window.location.href) instead of fetch(),
which bypasses CORS checks entirely.

KEY INSIGHT
================================================================================

When redirecting to an external OAuth provider, use direct navigation
(window.location.href) instead of fetch(). This bypasses CORS checks entirely
because it's treated as a normal page navigation, not an API call.
