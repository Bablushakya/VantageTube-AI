================================================================================
  HOW TO TEST
================================================================================

OPTION 1: POSTMAN (Recommended for GUI Testing)
  1. Start backend:
     cd vantagetube-ai/backend
     python -m uvicorn app.main:app --reload

  2. Open Postman
  3. Click "Import"
  4. Select: VantageTube_API_Collection.postman_collection.json
  5. Create environment with base_url = http://localhost:8000/api
  6. Click "Run" and select all requests
  7. Watch tests execute with visual feedback

OPTION 2: PYTHON SCRIPT (Automated Testing)
  1. Start backend:
     cd vantagetube-ai/backend
     python -m uvicorn app.main:app --reload

  2. Run tests (in another terminal):
     cd vantagetube-ai/backend
     python test_all_apis.py

  3. View results:
     cat test_results.json

OPTION 3: MANUAL CURL (Command Line)
  1. Start backend:
     cd vantagetube-ai/backend
     python -m uvicorn app.main:app --reload

  2. Test health:
     curl http://localhost:8000/health

  3. Register user:
     curl -X POST http://localhost:8000/api/auth/register \
       -H "Content-Type: application/json" \
       -d '{"email":"test@example.com","password":"TestPassword123!","confirm_password":"TestPassword123!","first_name":"Test","last_name":"User"}'

================================================================================
  COMMON ISSUES & SOLUTIONS
================================================================================

Issue: Connection refused
Solution: Start backend: python -m uvicorn app.main:app --reload

Issue: "Not authenticated"
Solution: Include Bearer token in Authorization header

Issue: "Invalid token"
Solution: Login again to get fresh token

Issue: YouTube API errors
Solution: Add YOUTUBE_API_KEY to .env

Issue: OpenAI API errors
Solution: Add OPENAI_API_KEY to .env

Issue: Supabase errors
Solution: Verify SUPABASE_URL and SUPABASE_KEY in .env

Issue: Port 8000 in use
Solution: Use different port: --port 8001
